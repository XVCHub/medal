API Reference

Decompile Luau bytecode back into readable source code via HTTP or WebSocket. Fast, simple, stateless.

2 Endpoints
Base64 Input Format
Luau Bytecode Type
Authentication

Authorization Header

every request must include a valid license key in the Authorization header. requests without it will be rejected.

⚠ include Authorization: <your_key> on every request. missing or invalid keys return 403 Forbidden.

Example Header
// HTTP header Authorization: ymjKH2O3BbO3bDSsKmpo3ek3vHxIWYLQfj0
Endpoints
POST /decompile HTTP · decompile bytecode, returns source as plain text
Request
field type description
body bytes raw request body — the bytecode encoded as base64
Authorization header your license key
Luau Example
local HttpService = game:GetService("HttpService") -- bytecode bytes as base64 string local encoded = base64_encode(bytecodeBytes) local result = request({ Url = "https://medaldecompiler.pages.dev/decompile", Method = "POST", Headers = { Authorization = "<your_key>" }, Body = encoded }) print(result.Body) -- decompiled source
JavaScript Example
const encoded = btoa(String.fromCharCode(...bytecodeBytes)) const res = await fetch("https://medaldecompiler.pages.dev/decompile", { method: "POST", headers: { Authorization: "<your_key>" }, body: encoded }) const source = await res.text() console.log(source)
Python Example
import base64, requests encoded = base64.b64encode(bytecode_bytes) r = requests.post( "https://medaldecompiler.pages.dev/decompile", headers={"Authorization": "<your_key>"}, data=encoded ) print(r.text)
Responses
200plain text — decompiled luau source code
400invalid bytecode — body is not valid base64
403invalid license — Authorization header missing or wrong
WS /decompile_ws WebSocket · batch decompile, async with message IDs

connect once, send multiple decompile jobs. each message gets an id back so you can match responses to requests — useful when firing many requests in parallel.

Message Format (send)
field type description
id string any string — echoed back in the response
encoded_bytecode string luau bytecode encoded as base64
Response Format (receive)
field type description
id string same id you sent
decompilation string decompiled luau source code
JavaScript Example
const ws = new WebSocket( "wss://medaldecompiler.pages.dev/decompile_ws", { headers: { Authorization: "<your_key>" } } ) ws.onopen = () => { ws.send(JSON.stringify({ id: "job_1", encoded_bytecode: btoa(String.fromCharCode(...bytecodeBytes)) })) } ws.onmessage = (e) => { const { id, decompilation } = JSON.parse(e.data) console.log(`[${id}]`, decompilation) }
Python Example
import asyncio, base64, json, websockets async def main(): uri = "wss://medaldecompiler.pages.dev/decompile_ws" headers = {"Authorization": "<your_key>"} async with websockets.connect(uri, extra_headers=headers) as ws: await ws.send(json.dumps({ "id": "job_1", "encoded_bytecode": base64.b64encode(bytecode).decode() })) resp = json.loads(await ws.recv()) print(resp["decompilation"]) asyncio.run(main())
Notes
ℹ Input must be base64. raw bytecode bytes will fail — always encode first.
ℹ Luau only. this decompiler handles luau bytecode. lua 5.1 bytecode is a different format.
ℹ WebSocket is better for bulk. for decompiling many scripts at once use /decompile_ws with multiple concurrent messages.
